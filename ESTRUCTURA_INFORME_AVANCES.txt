1. RESUMEN EJECUTIVO Y ESTADO ACTUAL
================================================================================

SIMC (Sistema de Monitoreo de Cultivos y Control de Humedad) es una aplicación integral 
que permite a agricultores gestionar fincas, cultivos, riegos y sensores IoT en tiempo real,
facilitando la toma de decisiones mediante alertas automáticas de humedad. El sistema está 
compuesto por un backend API REST desarrollado en Django y un frontend móvil en React Native/Expo.

- Propósito general del sistema: Sistema de monitoreo agrícola con control inteligente de riegos
- Porcentaje de ejecución Frontend: 80% (Interface principal completa, falta integración de gráficas avanzadas)
- Porcentaje de ejecución Backend: 85% (APIs funcionales, falta optimizaciones y reportes)
- Porcentaje de ejecución Base de Datos: 90% (Schema completo, relaciones establecidas)
- Estado general: En fase de pruebas y ajustes finales. API funcional, interfaces integradas

--- 3.2 ARQUITECTURA DEL SISTEMA ---

TIPO DE ARQUITECTURA:
Modelado Vista Controlador (MVC) con API REST - Arquitectura de capas separadas

DESCRIPCIÓN:
El sistema está dividido en tres capas independientes:
1. CAPA PRESENTACIÓN: Frontend React Native/Expo que proporciona interfaz de usuario móvil
2. CAPA LÓGICA DE NEGOCIO: Django REST Framework que procesa solicitudes y aplica validaciones
3. CAPA DATOS: SQLite que almacena la información persistente

Cada aplicación Django (usuarios, cultivos, sensores, mediciones, alertas) implementa:
- models.py: Definición de entidades y relaciones
- serializers.py: Conversión entre modelos y JSON
- views.py: ViewSets REST que exponen endpoints
- urls.py: Enrutamiento automático de DRF

FLUJO DE COMUNICACIÓN:
Cliente Mobile (React Native) → HTTP/REST (Axios) → Django API (puerto 8000) → ORM Django → SQLite

Autenticación: Bearer Token JWT en headers (Authorization: Bearer <token>)
Formato de datos: JSON en request/response
CORS: Habilitado para peticiones desde frontend en 172.31.0.183

  Archivo: MiAppTareas/app/services/api.ts
  Descripción: Cliente Axios configurado con interceptadores para manejo de tokens
  Características:
    - Almacenamiento seguro de token en AsyncStorage
    - Inyección automática de token en headers (Authorization: Bearer)
    - Renovación de token en caso de expiración
    - Limpieza al logout

Fragmento 3 - Validaciones Backend:
  Archivo: backend_cultivos/cultivos/serializers.py
  Descripción: Serializers con validaciones de datos de entrada
  Tipos de validación:
    - Campos requeridos y tipos de dato correctos
    - Validación de FK (finca existe y pertenece al usuario)
    - Validación de rangos (tamaño_hectareas > 0)

  Archivo: backend_cultivos/cultivos/views.py

  Lógica:

7. OBSERVACIONES Y CONCLUSIONES
================================================================================

LECCIONES APRENDIDAS:

1. ARQUITECTURA:
   - Separación clara entre Frontend/Backend facilitó paralelización del desarrollo
   - Django REST Framework fue excelente para CRUD rápido y escalable
   - React Native/Expo permitió prototipado rápido pero requisitos de optimización posteriores

2. DESARROLLO:
   - Testing temprano hubiera evitado bugs críticos en producción
   - Documentación en paralelo al código es esencial para mantenimiento
   - Code review debe ser parte del workflow desde inicio

3. COMUNICACIÓN:
   - Reuniones diarias de 15min evitaron muchos bloqueos
   - Documentación de decisiones arquitectónicas crucial para continuidad
   - Feedback de usuarios temprano invaluable para validación

4. GESTIÓN:
   - 4 semanas es tiempo ajustado pero viable con equipo dedicado
   - Buffer de tiempo siempre necesario para issues inesperados
   - Priorización clara (MVP vs Nice-to-have) evita scope creep

MEJORAS FUTURAS (v2.0):

1. FUNCIONALIDADES:
   - Exportación de reportes (PDF/Excel)
   - Dashboard de admin más detallado
   - Gráficas avanzadas con tendencias predictivas
   - Sistema de recomendaciones de riego
   - Integración con IoT devices específicos (Blynk, Arduino, Raspberry Pi)
   - App web (PWA) además de mobile
   - Integración con plataformas de pagos

2. TÉCNICO:
   - Migración a PostgreSQL (escalabilidad)
   - Caché distribuida (Redis)
   - Message queue (Celery) para tareas asincrónicas
   - Machine Learning para predicción de plagas
   - Websockets para notificaciones en tiempo real
   - Microservicios para módulos independientes

3. EXPERIENCIA:
   - Aplicación web responsiva (Next.js)
   - Soporte offline para app mobile
   - Multiidioma (i18n)
   - Temas Dark/Light mode
   - Accesibilidad mejorada (WCAG 2.1 AA)

PRÓXIMOS PASOS POST-LANZAMIENTO:
